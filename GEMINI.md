# Project Context: TanStack Filebase

## Overview
This project is a modern web application built with **React 19** and **Vite 7.2**, utilizing the **TanStack** ecosystem (specifically **TanStack Router**) for routing and state management patterns. It uses **TypeScript** for type safety and **Tailwind CSS v4** for styling.

## Key Technologies
- **Framework:** React 19
- **Build Tool:** Vite 7.2
- **Routing:** TanStack Router (File-based routing)
- **Styling:** Tailwind CSS v4 (configured via `@tailwindcss/vite`)
- **Language:** TypeScript
- **Linting & Formatting:** Biome
- **Testing:** Vitest, React Testing Library
- **Package Manager:** pnpm

## Architecture & Conventions

### Directory Structure
- **`src/routes/`**: Contains the file-based routes.
    - `__root.tsx`: The root layout component wrapping the entire application.
    - New files created here automatically generate routes (e.g., `about.tsx` -> `/about`).
- **`src/main.tsx`**: Application entry point. Sets up the `RouterProvider` and renders the app.
- **`src/routeTree.gen.ts`**: Automatically generated by TanStack Router. **Do not edit manually.**
- **`vite.config.ts`**: Vite configuration, including the TanStack Router plugin and Tailwind CSS.

### Routing (TanStack Router)
- **File-Based:** Routes are defined by creating files in `src/routes`.
- **Navigation:** Use the `<Link />` component from `@tanstack/react-router` for internal navigation.
- **Layouts:** `__root.tsx` acts as the main layout. Use `<Outlet />` to render child routes.

### Styling
- **Tailwind CSS:** Utility-first CSS framework.
- **Global Styles:** `src/styles.css` and `src/App.css`.

### Code Quality (Biome)
- This project uses **Biome** for both linting and formatting.
- Configuration is located in `biome.json`.
- **Formatter Settings:**
    - Indent: 2 spaces
    - Quotes: Single (JS/TS), Double (JSX/CSS)
    - Semicolons: As needed

## Development Workflow

### Setup
```bash
pnpm install
```

### Running the App
Start the development server (defaults to port 3000):
```bash
pnpm dev
```

### Building for Production
Build the application and run type checks:
```bash
pnpm build
```

### Testing
Run unit tests using Vitest:
```bash
pnpm test
```

### Linting & Formatting
Run Biome checks (linting and formatting verification):
```bash
pnpm biome check .
```
To automatically fix issues and format code:
```bash
pnpm biome check --write .
```

## Alias Configuration
- **`@`**: Maps to the `./src` directory.
  - Example: `import Button from '@/components/Button'`
